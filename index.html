ObjectiveService

A complete objective system for Roblox. Works on both Server and Client, supports cross-place saving, steps, timers, queues, and full event hooks — all through a single ModuleScript.

Client & Server
Cross-Place Save

01

Setup

Place the ModuleScript anywhere accessible (e.g. ReplicatedStorage). The module auto-detects if it's running on the Server or Client and behaves accordingly.

ℹ

The init() function is Client-only and must be called once before any other function. On the Server, no init is required — just require and call.

Client LocalScript

Luau — LocalScript
local svc = require(game.ReplicatedStorage.ObjectiveService)

svc.init({
    frame         = script.Parent.Main,
    titleLabel    = script.Parent.Main.Objectives.Obj.Objective1,
    tickIcon      = script.Parent.Main.Objectives.Obj.Tick,
    progressLabel = script.Parent.Main.Objectives.Progress.ProgressTxt,
})

Server Script

Luau — Script
local svc = require(game.ReplicatedStorage.ObjectiveService)

-- No init() needed on the server
svc.add(player, "Find the key.", nil, "Find the hidden key")
02

Quick Start

Simple objective

Luau
-- Text-only objective (no counter)
svc.add("Find the exit.", nil, "Reach the gate")

-- Complete it manually
svc.finish()

Objective with counter

Luau
svc.add("Collect crystals.", 5, "Crystals: ")

-- When player picks one up:
svc.increment()  -- 1/5, 2/5, 3/5...

-- Or set directly:
svc.update(3)    -- jumps to 3/5

-- Auto-completes when reaching 5/5!

With options

Luau
svc.add("Survive the wave!", nil, nil, {
    persistent  = true,     -- stays visible, never auto-hides
    onComplete  = function(obj)
        print("Done!", obj.title)
    end,
    data        = { reward = 500, type = "main" },
})
03

init(config)

Initializes the service and binds your UI elements. Must be called once on the client before anything else. All fields except frame are optional — if omitted, that UI feature is simply skipped.

Luau
svc.init({
    frame         = Frame,      -- REQUIRED: root container
    titleLabel    = TextLabel,  -- objective title text
    tickIcon      = ImageLabel, -- completion checkmark
    progressLabel = TextLabel,  -- counter / progress text
})
⚠

The frame must be a Frame (not a ScreenGui). The ScreenGui should stay enabled — the service controls visibility through the Frame's Visible property and child transparencies.

04

Objectives

add(title, total?, label?, options?) Both
Creates a new objective. Cancels any previous hide timer and shows the UI with a fade-in animation. If total is provided, shows a counter. On the Server, pass player as the first argument.
Parameters
titlestringThe objective text shown in titleLabel
totalnumber?Max count. When progress reaches this, auto-completes
labelstring?Text shown in progressLabel (e.g. "Crystals: " or fixed text)
optionstable?See options table below
update(progress) Both
Sets the current progress to a specific value. Re-shows the UI and restarts the hide timer. Automatically calls finish() when progress reaches total.
Parameters
progressnumberThe new progress value
increment() Both
Adds +1 to the current progress. Shorthand for update(progress + 1).
decrement() Both
Subtracts 1 from the current progress. Will not go below 0.
finish() Both
Marks the objective as complete. Re-shows the UI if hidden, waits 1s, shows the tick icon, waits 1.5s, then fades out and clears the objective. Fires onFinish and processes the queue if any.
cancel() Both
Cancels the current objective without showing the tick. Fades out and clears. Fires onCancel and processes the queue if any.
reset() Both
Resets progress to 0 and currentStep to 1 without canceling the objective. Re-shows the UI.

Options table

Luau
{
    persistent  = bool,   -- if true, UI never auto-hides
    timerMode   = bool,   -- if true, progressLabel shows countdown
    timer       = number, -- starts a countdown (seconds) immediately
    onComplete  = function(obj), -- callback only for this objective
    onCancel    = function(obj), -- callback if this objective is cancelled
    data        = {},      -- free custom data table, accessible via getData()
}

08

Steps System

Break an objective into sequential named steps. The progressLabel will automatically show which step the player is on.

Luau
svc.add("Complete the mission.")
svc.addStep("Enter the building")
svc.addStep("Find the key")
svc.addStep("Escape")

-- progressLabel shows: "Step 1/3: Enter the building"

svc.completeStep() -- "Step 2/3: Find the key"
svc.completeStep() -- "Step 3/3: Escape"
svc.completeStep() -- auto-calls finish()!
FunctionDescription
addStep(title)Appends a new step to the current objective
completeStep()Advances to the next step, or calls finish() on the last
getCurrentStep()Returns the current step index
getSteps()Returns the full steps array
09

Timer System

Add a countdown to an objective. When the timer reaches zero, finish() is called automatically.

Luau
-- Start with a timer immediately
svc.add("Defuse the bomb!", nil, "Time: ", {
    timer     = 30,
    timerMode = true,  -- shows "Time: 30s" in progressLabel
})

-- Or add timer after creation
svc.add("Survive!")
svc.setTimer(60)

-- Listen to each tick
svc.on("onTimerTick", function(remaining, total)
    if remaining == 10 then
        print("10 seconds left!")
    end
end)
FunctionDescription
setTimer(seconds)Starts a countdown. Auto-calls finish() at 0
stopTimer()Stops the countdown without finishing the objective
10

Queue System

Chain multiple objectives one after another. Each queued objective starts automatically 0.5s after the previous one finishes or is cancelled.

Luau
svc.queue("Find the entrance.", nil, "Find the gate")
svc.queue("Collect keys.", 3, "Keys: ")
svc.queue("Reach the portal.", nil, "Get to safety")

-- They'll play one by one automatically!
print(svc.getQueueSize()) -- 2 (first one starts immediately)
FunctionDescription
queue(title, total?, label?, options?)Adds to queue. If no active objective, starts immediately
clearQueue()Empties the queue (does not affect the current objective)
getQueue()Returns the queue array
getQueueSize()Returns the number of objectives waiting in queue
11

Save & Cross-Place

Persist objectives across sessions and across different Places in the same game using DataStore. All save functions are Server-only.

✦

Call load(player) on PlayerAdded to automatically restore the objective when a player enters a Place. The client receives a LoadSave remote and the onLoad event fires.

Cross-Place Example

Luau — Place 1 (Server)
-- Create and save progress
svc.add(player, "Collect gems.", 10, "Gems: ")
svc.update(player, 4)
svc.save(player)

-- Or teleport (auto-saves before teleporting)
svc.teleport(player, PLACE_ID_2)
Luau — Place 2 (Server)
Players.PlayerAdded:Connect(function(player)
    svc.load(player)  -- restores "Collect gems." at 4/10!
end)
FunctionDescription
save(player, placeId?)Saves current progress to DataStore. Optional placeId to save for a specific place
load(player, placeId?)Loads saved progress and sends it to the client. Optional placeId
teleport(player, placeId, opts?)Saves progress then teleports the player. Load on arrival
clearSave(player, placeId?)Deletes saved data from DataStore
hasSave(player, placeId?)Returns true if a save exists for the player
getState(player)Returns current in-memory state without hitting DataStore
ℹ

finish() and cancel() on the server automatically delete the DataStore entry — no need to call clearSave() manually when completing an objective.

12

Events

Subscribe to lifecycle events using on() and unsubscribe with off(). All events are Client-only.

Luau
local function onUpdate(obj)
    print(obj.title, obj.progress, obj.total)
end

svc.on("onUpdate", onUpdate)

-- Remove later
svc.off("onUpdate", onUpdate)
onAdd
Fires when a new objective is created
obj: table
onUpdate
Fires whenever progress changes
obj: table
onFinish
Fires after the completion animation ends
obj: table
onCancel
Fires when an objective is cancelled
obj: table
onTimerTick
Fires every second during a countdown
remaining: number, total: number
onTimerEnd
Fires when a timer reaches zero
obj: table
onStepComplete
Fires when a step is completed
stepIndex: number, obj: table
onQueue
Fires when a new item is added to the queue
queue: table
onLoad
Fires when a saved objective is loaded from DataStore
data: table

Server Functions

All client functions are available on the Server. Just pass a player as the first argument. The server fires a RemoteEvent to the target client automatically.

Server only
Client only
Both (signature differs)
Luau — Script (Server)
-- Same functions, player is first arg
svc.add(player, "Destroy targets.", 3, "Targets: ")
svc.increment(player)
svc.setTitle(player, "Almost there!")
svc.pause(player)
svc.finish(player)
svc.configure(player, { hideDelay = 5 })

Broadcast (All Players)

Send an objective to every player at once using the All variants.

Luau — Script (Server)
svc.addAll("Survive the wave!")
svc.updateAll(1)
svc.incrementAll()
svc.finishAll()
svc.cancelAll()
svc.showAll()
svc.hideAll()
svc.configureAll({ hideDelay = 4 })

16

Configure

Adjust global settings for the service at any time.

Luau
-- Change how long the UI stays visible
svc.setHideDelay(5)

-- Replace the entire tween animation
svc.setTweenInfo(
    TweenInfo.new(0.6, Enum.EasingStyle.Back, Enum.EasingDirection.Out)
)

-- Or configure multiple at once
svc.configure({
    hideDelay = 4,
})
